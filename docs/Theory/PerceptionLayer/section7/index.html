Section 7 - Wifi connection for the ESP32

Goals
  • Understand the steps necessary to set up and connect an ESP32 board to a Wi-Fi network.
  • Learn how to use the data structures from the Arduino WiFi library to create applications that utilize WiFi communication.
  • Illustrate different application scenarios where WiFi connection can be used.

Arduino core for the ESP32​

As previously mentioned, the ESP32 is an evolution of the ESP8266. In the lab, we have several NodeMCU-32S modules. The following list highlights some useful references if you are interested in the ESP32:

  • ESP32 - Manufacturer's page (link)
  • ESP-IDF Programming Guide (link)
  • Arduino core for the ESP32 (link)

In our case, we will focus on the Arduino core for the ESP32 (link), which contains all the information needed to program the ESP32 using the Arduino API (link).

Before starting, it is important to check the available libraries in the API, which can be viewed at the following link.

Arduino core for the ESP32: Wi-Fi API​

Since we are interested in analyzing how to connect the ESP32 to a WiFi network, the most relevant information can be found in the Wi-Fi API. Therefore, we will focus on giving a quick overview of this before providing some examples.

About the API, the website mentions the following:

About​

The Wi-Fi API provides support for the 802 11b/g/n protocol driver. This API includes:

  • Station mode (STA mode or Wi-Fi client mode)ESP32 connects to an access point
  • AP mode (aka Soft-AP mode or Access Point mode) Devices connect to the ESP32
  • Security modes (WPA, WPA2, WEP, etc).
  • Scanning for access points

As a basic starting point, it is good to remember the roles that the ESP32 can take within a Wi-Fi network:

  • Access Point (AP): In this mode, the ESP32 is configured as an access point (AP), allowing connection between other devices (Stations) on the network.

  • Station (STA): In this case, the ESP32 connects, like any other device, to the Wi-Fi network through an Access Point.

Connectivity test​

Before starting, the first thing we need to do is perform a basic test to verify the Wi-Fi functionality of the NodeMCU-32S module. So, follow these steps:

  1. Open the IDE IDE.
  2. Select the board (Tools > Board > ESP32 Arduino > NodeMCU-32s).

  1. Connect the NodeMCU-32S module to the PC.

  2. Select the port.

  1. In the example section (examples), look for examples related to NodeMCU-32S (Examples for NodeMCU-32S), and select WifiScan, thus: File > Examples > Wifi > WiFiScann)
  1. Compile and upload the example to the board. Unlike other boards, before proceeding with the firmware upload, you need to press and hold the 1OO button on the board and release it as soon as the message Connecting... appears in the upload console messages (for more information, see: Getting Started with the ESP32 Development Board).:
  1. Test the example by setting the serial terminal to the same value that appears in the program code (115200 in this case). If everything is correct, the serial monitor will display information about the available Wi-Fi networks as shown below:

Wifi libraries​

To make a connection using WiFi possible, an Access Point (AP) is necessary. An AP is a device that allows WiFi devices to connect to a wired network, as shown in the following figure:

To enable an Arduino board to connect to a WiFi network, the Arduino WiFi is used. This is made possible through the WiFi library (link), which is included in the Arduino IDE.

The following are the most commonly used classes in this library:

WiFi Class​

The WiFi class initializes the Ethernet library and network configuration. The following table shows some of the main methods:

MethodDescriptionSyntax
WiFi.begin()Initializes the network configuration of the WiFi library and provides the current statusWiFi.begin();
WiFi.begin(ssid);
WiFi.begin(ssid, pass);
WiFi.disconnect()Disconnects the WiFi board from the current networkWiFi.disconnect();
WiFi.status()Returns the connection status (WL_CONNECTED, WL_CONNECTION_LOST, WL_DISCONNECTED and WL_CONNECT_FAILED among others).WiFi.status();

For more information, please visit:

  1. Arduino WiFi library
  2. ESP8266 Arduino Core’s
  3. ESP8266 Arduino Core’s en español

IPAddress Class​

The IPAddress class provides information about the network configuration. The following table shows the methods of the WiFi class that are used with this class:

MethodDescriptionSyntax
WiFi.localIP()Gets the IP address of the WiFi boardWiFi.localIP();
WiFi.subnetMask()Gets the subnet mask of the WiFi boardWiFi.subnetMask();
WiFi.gatewayIP()Gets the gateway IP address of the WiFi boardWiFi.gatewayIP();

Server Class​

The Server class creates servers that can send and receive data from connected clients (programs running on other computers or devices). This class is the base class for WiFi Server. The following constructor is used for its instantiation:

ClassDescriptionConstructor syntax
WiFiServerCreates a server that listens for incoming connections on the specified portServer(port);

Below is a summary of some of the main methods associated with this class.

MethodDescriptionSyntax
begin()Instructs the server to start listening for incoming connectionsserver.begin();

Client Class​

The Client class creates clients that can connect to servers and send and receive data.

MethodDescriptionSyntax
WiFiClient()Creates a client that can connect to a specified IP address and Internet port as defined in client.connect().WiFiClient();
connected()Determines whether the client is connected or notclient.connected();

Due to the widespread use of the ESP8266, a WiFi library was created (which tries to maintain the philosophy of the original Arduino library) for this platform. For more information about this library, you can refer to the ESP8266WiFi library section of the ESP8266 Arduino Core API.

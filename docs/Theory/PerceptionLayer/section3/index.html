Framework Arduino​

When we talk about Arduino, we are not just referring to a specific microcontroller; in reality, we are talking about the most popular framework for microcontrollers today.

Arduino is an open-source electronics platform that combines software and hardware. As this platform is open hardware, it is possible to use the Arduino programming model to write code for any other Arduino-compatible platform (generic boards or those from other manufacturers).

The Arduino programming model is based on the Arduino API, which exposes a set of functions and structures (constants, variables, data types, objects, etc.) that enable interaction between the microcontroller and external hardware (sensors and actuators). The API information can found in Language Reference (link).

Arduino API basics functions​

Arduino API basics functions can be found online in Language Reference (link)

Digital Input and Output​

Input and Output functions, basically, allow two things:

  1. Configure the ports (pins) as digital inputs or outputs.
  2. Read from or write to these ports according to how they were configured.
FunctionInformation
digitalWrite()Description (link):
Write a HIGH or LOW value to a specific port.

Sintaxis:
digitalWrite(pin, value)

Parameters:
  • pin: Arduino port number.
  • value: Value written to the port (HIGH o LOW).
    • digitalRead()Description (link):
    Read the value from a specific port.

    Sintaxis:
    digitalRead(pin)

    Parameters:
  • pin: Arduino pin to be read.
  • Returned values: Port value to be read (HIGH o LOW).
    • pinMode()Description (link):
    Allows you to configure a port as input or output.

    Sintaxis:
    pinMode(pin, mode)

    Parameters:
  • pin: Arduino pin to be configured
  • mode: Modo: INPUT, OUTPUT o INPUT_PULLUP.

    • Analog input and output functions​

    They allow interaction (reading or writing) with ports that function as analog inputs or outputs. The following table summarizes some of these:

    FunctionInformation
    analogRead()Description (link):
    Reads the value from a specific analog port, returning an integer value associated with the voltage at that port.

    Sintaxis:
    analogRead(pin)

    Parameters:
  • pin: Analog port number (A0 – A5 for most of the boards) of Arduino


    • Returned values:
    Analog value read from the pin. The range will depend on the resolution of the analog-to-digital converter associated with the pin (0 – 1024 when the resolution is 10 bits or 0 – 4096 when the resolution is 12 bits).
    analogWrite()Description (link):
    Writes an analog value (by modifying the duty cycle of a PWM wave) to a port.

    Syntax:
    analogWrite(pin, value)

    Parameters:
  • pin: Arduino pin (denoted on the board with the ~ symbol) to which the value is written.
  • value: Duty cycle ranging from zero (always off) to 255 (always on).

    • Time Bases​

    These are functions used to create delays and generate timestamps in programs. There are several functions for this purpose, such as delay(), delayMicroseconds(), micros() y millis(). The following table describes the function delay(), which was the first one implemented in the above examples:

    FunctionInformation
    delay()Description (link):
    Stops the program for a specified amount of time (in milliseconds) given as a parameter.

    Syntax:
    delay(ms)

    Parameters:
  • ms: Number of milliseconds to stop the program.

    • Serial port interaction functions​

    These are functions used for configuring and interacting with the serial port (link). The following table shows some commonly used functions:

    FunctionDescription
    Serial.begin()Configures the serial transmission speed (bits per second = baud).

    Syntax:
    Serial.begin(speed)

    Parameters:
  • speed: Speed transmission
    • Serial.print()Prints data to the serial port in ASCII format.

    Syntax:
    Serial.print(val)
    Serial.print(val, format)

    Parameters:
  • val: Value to print. The value can be of any type.
  • format: ASCII representation format (DEC, HEX, OCT o BIN).
    • Serial.available()Gets the number of bytes (characters) available to read from the serial port. These are data that have already arrived and have been stored in the serial receive buffer (which holds 64 bytes).

    Syntax:
    Serial.available()

    Returned values: Number of bytes available to read.
    Serial.read()Reads a piece of data coming through the serial port.

    Syntax:
    Serial.read()

    Returned values: First byte of the available serial data (or -1 if no data is available). The data type read is int.

    Pinout

    As mentioned in previous classes, the starting point is always to know the pinout diagram of the development board. This is shown below.:

    The pinout map of NodeMCU ESP8266 is:

    • 17 GPIO’s
    • SPI
    • I2C (implemented on software)
    • I2S interfaces with DMA
    • UART
    • 10-bit ADC

    Additionally, the pin numbering scheme of the NodeMCU board (for example, the pin marked as D0) is different from the numbering scheme of the ESP8266 (internal scheme). For instance, the D0 pin on the board is mapped to the internal GPIO16. The recommendation is to use the external numbering scheme.

    The following table shows the equivalence between the board numbering and the internal numbering:

    NodeMCU pinESP8266 pin
    D0GPIO16
    D1GPIO5
    D2GPIO4
    D3GPIO0
    D4GPIO2
    D5GPIO14
    D6GPIO12
    D7GPIO13
    D8GPIO15
    D9GPIO3
    D10GPIO1
    D11GPIO9
    D12GPIO10

    According to the previous table, it is possible to code sketches using either of the two nomenclatures. To understand this, observe the following code snippet where the D2 pin (GPIO4) is used as an output.

    • NodeMCU Nomenclature

      pinMode(D2, OUTPUT);
      digitalWrite(D2, HIGH);

    • ESP8266 Nomenclature

      pinMode(4, OUTPUT);
      digitalWrite(4, HIGH);

    It is important to take into account that the D0 pin (GPIO16) can be used only for GPIO read/write. It does not support open-drain/interrupt/PWM/I²C or 1-Wire. To understand better more about the pinout map, please visit the page ESP8266 Pinout Reference: Which GPIO pins should you use? (link)
